I. Objective  
The objective of this task is to implement a robust, tabular Master Monitoring Grid Module  inside qTASK. This module will 
aggregate data across all 15 system phases into a single comprehensive table, equipping project managers with multi -
column filtering capabilities to generate instant progress and target -date tracking reports.  
 
 
II. Specifications  
A. Frontend & User Interface (UI) Requirements  
The OJT team must construct a clean, highly scannable tabular grid view based on the layout and filtering paradigms 
found in advanced tracking systems:  
 
A. Core Monitoring Table Columns  
The data grid must present a flat table structure featuring the exact columns below:  
1. Project:  Name of the parent project (e.g., BANNER , NOVO , etc.).  
2. Task Title:  The full descriptive name of the task block.  
3. Phase:  The current step out of the 15 pipeline phases (e.g., In Progress , QA Testing , etc.).  
4. Severity:  The priority level of the task card (e.g., CRITICAL / SHOWSTOPPER , HIGH , MEDIUM ). 
5. Status:  The current health state of the activity (e.g., Active , Blocked , Bug Fixing , Passed ). 
6. Assignee:  The full name and avatar badge of the assigned OJT member.  
7. Target Date:  The definitive deadline date stamped on the task card.  
8. Progress:  A completion metric displaying either a percentage tracker or checklist ratio (e.g., 8/10 or 80%).  
 
B. Live Multi -Filter Control Strip  
Directly above the monitoring table, a unified control bar must feature separate dropdown filter selectors for each  
column:  
• [ Filter by Project: ALL v ]  
• [ Filter by Assignee: ALL v ]  
• [ Filter by Status: ALL v ]  
• [ Filter by Phase: ALL v ]

C. Technical Stack Recommendations  
• Frontend Grid Libraries:  It is strongly recommended to use a data grid package like AG Grid (Community 
Edition) , TanStack Table (React Table) , or standard Bootstrap DataTables . These libraries come out -of-the-
box with built -in multi -column sorting, row filtering, and pagination styles.  
• Export Component:  Add a simple [ Export to Excel / CSV ] utility button next to the filter controls to let managers 
download their filtered monitoring snapshots instantly.  
 
 
III. Definition of Done (DoD)  
This module shall be considered officially complete when:  
1. The grid properly pulls and correctly populates all 8 mandated columns from the main tasks database.  
2. Combining multiple filter selections concurrently (e.g., Project: BANNER  + Status: Blocked  + Severity: Critical ) yields 
instant, mathematically accurate row results.  
3. Clicking column headers correctly triggers descending and ascending rows sorting sequences.  
4. The overall text padding, row layout separators, and filter dropdown actions mirror the professional and fast filtering 
interface found in TickTick.
